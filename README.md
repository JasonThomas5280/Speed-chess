# Speed Chess

A mobile-first, installable **PWA** for fast chess against a bot. Bullet & blitz
vs [Stockfish](https://stockfishchess.org/) (WASM), running **entirely in the
browser** — no backend, no accounts, works **offline**. $0 to run on any static
host.

## Features

- ♟️ **Tap _and_ drag** to move, with optimistic rendering — the piece lands the
  instant your finger lifts, legality is synchronous (chess.js).
- ⚡ **Premove** during the bot's think time (native chessground premove).
- 🤖 **Stockfish 16** (single-threaded WASM) in a Web Worker, so the UI never
  freezes. Friendly **1–10 strength** dial maps to Skill Level + a snappy,
  blitz-capped think time.
- ⏱️ **Big high-contrast clocks** with a low-time color shift + pulse. The clock
  runs independently of the engine, so a slow reply never freezes your timer;
  flag (time-out) detection lives in the clock tick.
- 🎚️ **Time controls**: 1+0, 2+1, 3+0, 3+2, 5+0, plus custom — with Fischer
  increment.
- 🎨 Board themes, sound cues (Web Audio, synthesized — no audio files), and
  haptics (`navigator.vibrate`).
- 🧘 **Deliberate friction**: a clean result screen with one-tap rematch — but no
  auto-requeue, and a neutral "games today" counter. Treats you like an adult.
- 📲 **Installable + offline**: manifest, service worker, precached engine wasm.

## Stack

React + Vite + TypeScript · Tailwind CSS · [chessground](https://github.com/lichess-org/chessground)
(board UI) · [chess.js](https://github.com/jhlywa/chess.js) (rules / source of
truth) · Stockfish WASM (bot) · vite-plugin-pwa (Workbox) · localStorage
(settings, stats).

chess.js is the **single source of truth** for game state; chessground is display
only.

## Develop

```bash
npm install      # a prebuild step copies the Stockfish worker + wasm into public/engine/
npm run dev      # http://localhost:5173
npm run build    # type-check + production build into dist/
npm run preview  # serve the production build locally
```

> The Stockfish worker + `.wasm` are copied from `node_modules/stockfish` into
> `public/engine/` by `scripts/copy-stockfish.mjs` (wired into `predev` /
> `prebuild`). They're precached by the service worker so offline play works.
> Icons are generated by `scripts/gen-icons.mjs` (already committed).

## Deploy

Any static host with HTTPS and `application/wasm` for `.wasm` (required for PWA):
**Cloudflare Pages / Netlify / Vercel** all do this automatically. Build command
`npm run build`, output directory `dist/`.

## Project layout

```
src/
  App.tsx              # game loop: wires board <-> chess.js <-> engine, owns the clock
  components/          # Board, Clock, Controls, ResultScreen, Settings, StatsBadge
  engine/
    chessState.ts      # chess.js wrapper (dests, check, result)
    useStockfish.ts    # worker lifecycle, FEN -> bestmove, skill mapping
  lib/                 # storage, time, sound, haptics
  styles/index.css     # tailwind + chessground board themes & highlights
scripts/
  copy-stockfish.mjs   # vendors the engine into public/engine/
  gen-icons.mjs        # generates PWA icons (zero deps)
```
